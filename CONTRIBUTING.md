# Contributing to ProxyMaster

Thank you for considering contributing to ProxyMaster! This document outlines how you can contribute to make this project better.

## Ways to Contribute

1. **Report Bugs**: If you find a bug, please create an issue with detailed steps to reproduce it
2. **Suggest Features**: Have ideas for new features? Open an issue with the tag "enhancement"
3. **Submit Pull Requests**: Implement new features or fix bugs and submit pull requests
4. **Improve Documentation**: Help us improve our documentation with clearer explanations or examples
5. **Share Success Stories**: If you're using ProxyMaster effectively, we'd love to hear about it!

## Development Setup

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/proxy-scraper.git`
3. Create a branch for your changes: `git checkout -b feature-or-fix-name`
4. Install development dependencies: `pip install -r requirements.txt`

## Pull Request Guidelines

1. Update the README.md with details of changes if appropriate
2. Update the documentation if you're changing functionality
3. Make sure your code follows the existing style
4. Write tests for new functionality
5. Run all tests before submitting
6. Keep pull requests focused on a single topic

## Code Style

- Follow PEP 8 guidelines for Python code
- Use meaningful variable and function names
- Include docstrings for functions and classes
- Comment complex sections of code

## Testing

- Add tests for new functionality
- Run all tests before submitting changes
- Make sure existing tests pass with your changes

## Communication

- Use clear and descriptive titles for issues and pull requests
- Be respectful and considerate in all communications
- Provide as much context as possible when reporting issues

Thank you for contributing to ProxyMaster!
